# Tags vs. Chips

<ImageGrid
    images={[TagsNChipsTag, TagsNChipsChips]}
    captionsUnderImages={[`Tags — static and non-interactive.`, `Chips — contextual and interactive.`]}
/>

The name "tag" may sometimes be used for another UI component "chip", but they serve different purposes. Tags are static and non-interactive. They are commonly used as status indicators or something to capture users’ attention. Chips are versatile, contextual, and interactive. They are used for filtering or suggested actions. Chips may also contain auxiliary elements like leading icons or trailing remove icons.

This guide deals specifically with the design of tags. For chips, please refer to [Material Design's guidelines](https://m2.material.io/components/chips).

# Common Use Cases

Tags are nearly exclusively seen in [list items](/patterns/lists) to annotate a status such as alarms, warnings, active, and acknowledged. It can also be used to call for visual attention (e.g., “Update Required”).

<ImageGrid
    images={[InList, InAlarms, InList2, InTimestamp, Inline, InTables]}
    caption={`Tags used inside different components.`}
/>

Some designers may choose to use icons to depict statuses, but tags require less “guesswork” from users. Although there are a few "[universal icons](/style/icon-guidelines#common-universal-icons)" that's instantly recognizable and should stay as icons, If users are not expected to frequently interact with your application, they should not be forced to memorize all the rare icons and their meanings — and tags are wonderful for avoiding this situation.

<ImageGrid
    images={[GoodIcon]}
    captionsUnderImages={[`DO: Use universal icons. They are long-established and widely recognizable.`]}
/>

<ImageGrid
    images={[BadIcon, GoodTag]}
    captionsUnderImages={[`DONT: Do not invent an icon for a rare status.`, `DO: Use tags instead.`]}
/>

# Variants

Brightlayer UI tags have three different variants: filled, shaded, and outlined.

Similar to how [button styles](/patterns/buttons#styles) can be used to guide users’ attention, tag variants can also be used to convey different urgencies and visual hierarchies.

# Colors

Tag colors should be consistent with status colors throughout the application. For recommendations on how colors are used across Brightlayer UI applications, please refer to our [color guides](/style/color).

In the dark theme, the tag colors should reflect the respective theme colors appropriately.

<ImageGrid images={[DarkTheme]} captionsUnderImages={[`Tags in the light and dark themes.`]} />

Always double check the color choices to ensure that the tag meets the [accessibility requirements](/design/accessibility).

<ImageGrid
    images={[YellowTagsDont, YellowTags]}
    captionsUnderImages={[
        `DONT: Do not use color combinations that make tags hard to read. This is especially important for light colors such as yellow and orange.`,
        `DO: Always select colors that makes the text readable.`,
    ]}
/>

# Labels

To ensure users can easily read the tag while keeping the tag size as small as possible, the font style of tags is 10dp, all caps, and bolded. Do not use lowercase letters on tags, as they are less legible at a small font size.

<ImageGrid images={[LowerCase]} captionsUnderImages={[`DONT: Do not use lowercase letters in tags.`]} />

## Text-Only

Use only texts inside a tag. Do not use additional elements such as leading icons and trailing remove icons.

<ImageGrid
    images={[TextOnly]}
    captionsUnderImages={[
        `DONT: Do not include anything in tags other than texts. Chips should be used in this situation.`,
    ]}
/>

## Short Labels

Keep the label short. Ideally, the tag label should contain only 1-2 words that form an adjective.

<ImageGrid
    images={[LongWords]}
    captionsUnderImages={[
        `AVOID: Avoid using more words than necessary. "Updates available" is shorter and can convey the same message.`,
    ]}
/>

Be careful when designing tags with a fixed width, as the tag label can overflow when translated into a new language.

<ImageGrid images={[I18N]} captionsUnderImages={[`CAUTION: Fixed-width tags may cause labels to overflow.`]} />

# Multiple Tags

List items can be flagged with more than one tag. Avoid assigning different tags with the same or similar variants and colors.

<ImageGrid
    images={[MultiTagSameColor]}
    captionsUnderImages={[
        `AVOID: Avoid using the same variant and color for different tags, as it becomes difficult to distinguish tags from a glance.`,
    ]}
/>

Order tags by frequency to minimize the chance of tags shifting horizontally in a list.

<ImageGrid
    images={[MultiTagShiftingDont, MultiTagShiftingDo]}
    captionsUnderImages={[
        `AVOID: Avoid "tag shifting" in lists.`,
        `DO: Minimize users' effort to scan through tags by placing the most frequently used tag to the right.`,
    ]}
/>

If you are running out of horizontal spaces, such as when designing for mobile devices, there are a few ways to reposition tags to a more appropriate place.

<ImageGrid
    images={[MultiTagInfoLine, MultiTagEllipsis, MultiTagWrap, MultiTagWrapInline]}
    captionsUnderImages={[
        `Tags can be placed next to list item texts.`,

        `Tags can be wrapped to a new line as "inline blocks".`,
        `DONT: Never wrap within a tag.`,
    ]}
/>
